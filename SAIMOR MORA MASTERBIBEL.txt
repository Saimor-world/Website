Ein lebendiger Raum, der versteht:
* Was du brauchst

* Was du suchst

* Was du vergessen hast

* Wo Zusammenhänge sind

SAIMÔR ist ein „Semantic Operating System“ (SOS).
Es ist kein Tool, sondern eine hohe Abstraktionsschicht über allen Tools.
Prinzipien:
                                                                                                                                                * Das System führt den Nutzer.

                                                                                                                                                * Das System ordnet automatisch.

                                                                                                                                                * Das System ist ein Raum, nicht eine Liste.

                                                                                                                                                * Das System ist semantisch, nicht technisch.

                                                                                                                                                * Das System ist visuell verständlich.

                                                                                                                                                * Das System komponiert Informationen.

                                                                                                                                                * Das System ist ein lebender Organismus.

⭐ 3.2 Gesamtsystem – Makro-Architektur
SAIMÔR besteht aus 6 Subsystemen:
1. Playground OS (Frontend)
2. Pane System (Fenster)
3. Dock (Control Bar)
4. Backend Core (Daten, Events)
5. Semantic Graph (Qdrant / Embedding)
6. MÔRA Intelligence Layer (LLM + Nested Learning)


Datenfluss (vereinfacht):
User → Playground → Pane → API → Core → Semantic Layer → Awareness → Orb Cursor → User

⭐ 9.6 Hard Interaction Rules (Gold-Regeln)
Regel 1
Das Universum bleibt IMMER sichtbar.
 Keine Subpages, keine neuen Screens.
Regel 2
Alles entsteht aus Orb oder Dock.
Regel 3
Cursor zeigt Intent.
 Wenn der Cursor sich bewegt → MÔRA hat eine Absicht.
Regel 4
Panes dürfen sich NIE Vollbild erzwingen.
 Immer OS-hafte Fenster.
Regel 5
Die KI darf den Nutzer niemals überfahren.
 Sie zeigt, schlägt vor, erklärt — nicht diktieren.
Regel 6
Kein Element darf hart animieren.
 Alles ist weich, organisch, lichtbasiert.
Regel 7
Notifikationen sind heilig:
Klein, subtil, präzise.
